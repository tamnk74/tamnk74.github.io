Social Login Setup (Google & GitHub)

September 1, 2025 ¬∑ 3 min read

This document explains how to set up Google and GitHub OAuth providers in your Supabase project.

1. Google OAuth Setup‚Äã

  1. Create Google OAuth App:

    • Go to Google Cloud Console
    • Create a new project or select existing one
    • Enable Google+ API
    • Go to "Credentials" ‚Üí "Create Credentials" ‚Üí "OAuth 2.0 Client IDs"
    • Choose "Web application"
    • Add authorized redirect URIs: 
      https://your-project-ref.supabase.co/auth/v1/callback

  2. Configure in Supabase:

    • Go to your Supabase project dashboard
    • Navigate to "Authentication" ‚Üí "Providers"
    • Enable "Google"
    • Add your Google Client ID and Client Secret
    • Save configuration

2. GitHub OAuth Setup‚Äã

  1. Create GitHub OAuth App:

    • Go to GitHub Settings ‚Üí Developer settings ‚Üí OAuth Apps
    • Click "New OAuth App"
    • Fill in the details:
      • Application name: Your app name
      • Homepage URL: Your app URL
      • Authorization callback URL: 
        https://your-project-ref.supabase.co/auth/v1/callback

  2. Configure in Supabase:

    • Go to your Supabase project dashboard
    • Navigate to "Authentication" ‚Üí "Providers"
    • Enable "GitHub"
    • Add your GitHub Client ID and Client Secret
    • Save configuration

Frontend Components‚Äã

  • Login Form (src/features/auth/login-form/ui.tsx):

    • Google login button with brand colors
    • GitHub login button
    • Loading states and error handling
    • Disabled state during authentication

  • OAuth Callback (src/app/[locale]/auth/callback/route.ts):

    • Handles OAuth redirects from providers
    • Exchanges authorization codes for sessions
    • Redirects with success/error messages

User Experience‚Äã

  1. Login Flow:

    • User clicks Google/GitHub button
    • Redirected to provider's authorization page
    • After approval, redirected back to /auth/callback
    • Session established and redirected to home page

  2. Error Handling:

    • Provider authentication failures
    • Network errors
    • Invalid callback requests
    • User-friendly error messages

üîí Security Features‚Äã

  • PKCE Flow: Uses Proof Key for Code Exchange for security
  • State Validation: Prevents CSRF attacks
  • Secure Redirects: Only allows authorized redirect URLs
  • Session Management: Proper cookie handling and session storage

üé® UI/UX Improvements‚Äã

  • Brand Colors: Google (red) and GitHub (gray) themed buttons
  • Loading States: Buttons disabled during authentication
  • Clear Messaging: Success and error feedback
  • Responsive Design: Works on mobile and desktop

üìù Usage Notes‚Äã

  • Make sure to configure your OAuth apps with the correct callback URLs
  • Test both providers in development and production environments
  • The callback route handles both success and error cases
  • Users will be redirected to the home page after successful login

üêõ Troubleshooting‚Äã

  1. "Invalid redirect URI": Check your OAuth app callback URLs
  2. "Client ID not found": Verify Supabase provider configuration
  3. "Authentication failed": Check provider app permissions and status
  4. Session not persisting: Verify cookie settings and domain configuration
