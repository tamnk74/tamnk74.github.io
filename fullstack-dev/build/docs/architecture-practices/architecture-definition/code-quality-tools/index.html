Code Quality Tools

This guide provides comprehensive setup and usage instructions for code quality tools in NestJS microservices and Next.js microfrontend projects.

Overview​

Code quality tools are essential for maintaining consistent, secure, and maintainable code across a microservices architecture. This guide covers automated code style verification, quality analysis, and integration with development workflows.

Core Quality Tools​

ESLint Configuration​

ESLint provides automated code style verification and catches potential bugs and anti-patterns.

Project-wide ESLint Setup​

    "format": "prettier --write .",
    "format:check": "prettier --check .",

    "quality:check": "npm run lint:check && npm run format:check && npm run type-check"

  }

Quality Gates and Automation​

GitHub Actions Quality Workflow​

# .github/workflows/quality.yml
name: Code Quality

on:
  pull_request:
    branches: [main, develop]
  push:
    branches: [main, develop]

jobs:
  quality:
    runs-on: ubuntu-latest
    
    steps:
      - name: Checkout code
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Type checking
        run: npm run type-check

      - name: Lint checking
        run: npm run lint:check

      - name: Format checking
        run: npm run format:check

      - name: Security audit
        run: npm audit --audit-level high

      - name: Run tests with coverage
        run: npm run test:coverage

      - name: SonarQube Scan
        uses: sonarqube-quality-gate-action@master
        env:
          SONAR_TOKEN: ${{ secrets.SONAR_TOKEN }}
          SONAR_HOST_URL: ${{ secrets.SONAR_HOST_URL }}

      - name: Upload coverage to Codecov
        uses: codecov/codecov-action@v3
        with:
          file: ./coverage/lcov.info
          flags: unittests
          name: codecov-umbrella

Quality Gate Configuration​

// scripts/quality-gate.ts
import { execSync } from 'child_process';
import { readFileSync } from 'fs';

interface QualityMetrics {
  coverage: number;
  duplicatedLines: number;
  maintainabilityRating: string;
  reliabilityRating: string;
  securityRating: string;
}

class QualityGate {
  private readonly thresholds = {
    coverage: 80,
    duplicatedLines: 3,
    maintainabilityRating: 'A',
    reliabilityRating: 'A',
    securityRating: 'A',
  };

  async checkQuality(): Promise<void> {
    console.log('🔍 Running quality gate checks...');

    // Check test coverage
    await this.checkCoverage();
    
    // Check linting
    await this.checkLinting();
    
    // Check formatting
    await this.checkFormatting();
    
    // Check type safety
    await this.checkTypes();
    
    // Check security vulnerabilities
    await this.checkSecurity();

    console.log('✅ All quality gate checks passed!');
  }

  private async checkCoverage(): Promise<void> {
    try {
      execSync('npm run test:coverage', { stdio: 'inherit' });
      
      const coverageData = JSON.parse(
        readFileSync('coverage/coverage-summary.json', 'utf8')
      );
      
      const totalCoverage = coverageData.total.lines.pct;
      
      if (totalCoverage < this.thresholds.coverage) {
        throw new Error(
          `Coverage ${totalCoverage}% is below threshold ${this.thresholds.coverage}%`
        );
      }
      
      console.log(`✅ Coverage: ${totalCoverage}%`);
    } catch (error) {
      console.error('❌ Coverage check failed:', error.message);
      process.exit(1);
    }
  }

  private async checkLinting(): Promise<void> {
    try {
      execSync('npm run lint:check', { stdio: 'inherit' });
      console.log('✅ Linting passed');
    } catch (error) {
      console.error('❌ Linting failed');
      process.exit(1);
    }
  }

  private async checkFormatting(): Promise<void> {
    try {
      execSync('npm run format:check', { stdio: 'inherit' });
      console.log('✅ Formatting passed');
    } catch (error) {
      console.error('❌ Formatting failed');
      process.exit(1);
    }
  }

  private async checkTypes(): Promise<void> {
    try {
      execSync('npm run type-check', { stdio: 'inherit' });
      console.log('✅ Type checking passed');
    } catch (error) {
      console.error('❌ Type checking failed');
      process.exit(1);
    }
  }

  private async checkSecurity(): Promise<void> {
    try {
      execSync('npm audit --audit-level high', { stdio: 'inherit' });
      console.log('✅ Security audit passed');
    } catch (error) {
      console.error('❌ Security audit failed');
      process.exit(1);
    }
  }
}

// Run quality gate
const qualityGate = new QualityGate();
qualityGate.checkQuality().catch(console.error);

// src/monitoring/quality-metrics.service.ts
import { Injectable } from '@nestjs/common';
import { Cron, CronExpression } from '@nestjs/schedule';
import { HttpService } from '@nestjs/axios';

@Injectable()
export class QualityMetricsService {
  constructor(private readonly httpService: HttpService) {}

  @Cron(CronExpression.EVERY_DAY_AT_MIDNIGHT)
  async collectQualityMetrics(): Promise<void> {
    const metrics = await this.getSonarQubeMetrics();
    const coverage = await this.getCoverageMetrics();
    
    await this.sendMetricsToMonitoring({
      ...metrics,
      ...coverage,
      timestamp: new Date().toISOString(),
    });
  }

  private async getSonarQubeMetrics(): Promise<any> {
    const response = await this.httpService
      .get(`${process.env.SONAR_HOST_URL}/api/measures/component`, {
        params: {
          component: process.env.SONAR_PROJECT_KEY,
          metricKeys: 'coverage,duplicated_lines_density,sqale_rating,reliability_rating,security_rating',
        },
        headers: {
          Authorization: `Bearer ${process.env.SONAR_TOKEN}`,
        },
      })

Best Practices​

Code Quality Standards​

  1. Maintain High Coverage: Keep test coverage above 80%
  2. Fix Issues Promptly: Address lint errors and warnings immediately
  3. Use Type Safety: Leverage TypeScript's strict mode
  4. Regular Audits: Run security audits regularly
  5. Consistent Formatting: Use automated formatting tools

Tool Configuration​

  1. Shared Configuration: Use consistent tool configurations across projects
  2. IDE Integration: Configure IDEs for optimal development experience
  3. Automated Enforcement: Use Git hooks and CI/CD for quality gates
  4. Regular Updates: Keep tools and configurations up to date

Team Collaboration​

  1. Quality Reviews: Include code quality in PR reviews
  2. Knowledge Sharing: Share quality best practices across teams
  3. Continuous Improvement: Regularly review and improve quality processes
  4. Tool Training: Ensure team members understand quality tools
