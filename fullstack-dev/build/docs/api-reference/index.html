API Reference

This document provides comprehensive API reference for the Next.js for Production application.

Authentication APIs​

POST /api/auth/signin​

Sign in a user with email and password.

Request Body:

{
  "email": "user@example.com",
  "password": "securepassword"
}

Response:

{
  "user": {
    "id": "uuid",
    "email": "user@example.com",
    "name": "User Name",
    "role": "user"
  },
  "session": {
    "access_token": "jwt_token",
    "expires_at": 1234567890
  }
}

POST /api/auth/signup​

Create a new user account.

Request Body:

{
  "email": "user@example.com",
  "password": "securepassword",
  "name": "User Name"
}

POST /api/auth/signout​

Sign out the current user.

Response:

{
  "message": "Signed out successfully"
}

User Management APIs​

GET /api/users​

Get list of users (Admin only).

Query Parameters:

  • page (number): Page number (default: 1)
  • limit (number): Items per page (default: 10)
  • search (string): Search query

Response:

{
  "users": [
    {
      "id": "uuid",
      "email": "user@example.com",
      "name": "User Name",
      "role": "user",
      "created_at": "2024-01-01T00:00:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 10,
    "total": 100,
    "totalPages": 10
  }
}

GET /api/users/[id]​

Get user by ID.

Response:

{
  "id": "uuid",
  "email": "user@example.com",
  "name": "User Name",
  "role": "user",
  "profile": {
    "avatar": "url",
    "bio": "User bio"
  }
}

PUT /api/users/[id]​

Update user information.

Request Body:

{
  "name": "Updated Name",
  "profile": {
    "bio": "Updated bio"
  }
}

Calendar APIs​

GET /api/calendar/events​

Get calendar events.

Query Parameters:

  • start (string): Start date (ISO 8601)
  • end (string): End date (ISO 8601)
  • category (string): Event category filter

Response:

{
  "events": [
    {
      "id": "uuid",
      "title": "Meeting",
      "description": "Team sync",
      "start": "2024-01-01T10:00:00Z",
      "end": "2024-01-01T11:00:00Z",
      "category": "meeting",
      "recurring": {
        "type": "weekly",
        "interval": 1,
        "end_date": "2024-12-31T23:59:59Z"
      }
    }
  ]
}

POST /api/calendar/events​

Create a new calendar event.

Request Body:

{
  "title": "New Event",
  "description": "Event description",
  "start": "2024-01-01T10:00:00Z",
  "end": "2024-01-01T11:00:00Z",
  "category": "meeting",
  "recurring": {
    "type": "weekly",
    "interval": 1
  }
}

PUT /api/calendar/events/[id]​

Update an existing event.

DELETE /api/calendar/events/[id]​

Delete an event.

Library APIs​

GET /api/library/books​

Get library books with search and filtering.

Query Parameters:

  • q (string): Search query
  • author (string): Filter by author
  • category (string): Filter by category
  • page (number): Page number
  • limit (number): Items per page

Response:

{
  "books": [
    {
      "id": "uuid",
      "title": "Book Title",
      "author": "Author Name",
      "isbn": "978-0000000000",
      "category": "Technology",
      "description": "Book description",
      "cover_url": "https://example.com/cover.jpg",
      "available": true,
      "rating": 4.5,
      "reviews_count": 10
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 10,
    "total": 100,
    "totalPages": 10
  },
  "facets": {
    "categories": [
      {"name": "Technology", "count": 25},
      {"name": "Science", "count": 15}
    ],
    "authors": [
      {"name": "Author Name", "count": 5}
    ]
  }
}

POST /api/library/books​

Add a new book to the library.

Request Body:

{
  "title": "New Book",
  "author": "Author Name",
  "isbn": "978-0000000000",
  "category": "Technology",
  "description": "Book description",
  "cover_url": "https://example.com/cover.jpg"
}

GET /api/library/books/[id]​

Get book details by ID.

PUT /api/library/books/[id]​

Update book information.

DELETE /api/library/books/[id]​

Remove book from library.

Search APIs​

GET /api/search​

Global search across all content types.

Query Parameters:

  • q (string): Search query (required)
  • types (string): Comma-separated content types (books,events,users)
  • page (number): Page number
  • limit (number): Items per page

Response:

{
  "results": [
    {
      "type": "book",
      "id": "uuid",
      "title": "Matching Book",
      "highlight": "Highlighted text...",
      "score": 0.95
    },
    {
      "type": "event",
      "id": "uuid",
      "title": "Matching Event",
      "highlight": "Highlighted text...",
      "score": 0.87
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 10,
    "total": 25,
    "totalPages": 3
  },
  "facets": {
    "types": [
      {"name": "books", "count": 15},
      {"name": "events", "count": 10}
    ]
  }
}

GET /api/search/suggestions​

Get search suggestions for auto-complete.

Query Parameters:

  • q (string): Partial search query
  • limit (number): Number of suggestions (default: 5)

Response:

{
  "suggestions": [
    "JavaScript fundamentals",
    "JavaScript frameworks",
    "JavaScript testing"
  ]
}

Health Check APIs​

GET /api/health​

Application health check endpoint.

Response:

{
  "status": "ok",
  "timestamp": "2024-01-01T00:00:00Z",
  "uptime": 12345,
  "checks": {
    "database": "ok",
    "redis": "ok",
    "elasticsearch": "ok"
  }
}

GET /api/health/detailed​

Detailed health check with service statuses.

Response:

{
  "status": "ok",
  "timestamp": "2024-01-01T00:00:00Z",
  "services": {
    "supabase": {
      "status": "ok",
      "response_time": 45,
      "last_check": "2024-01-01T00:00:00Z"
    },
    "elasticsearch": {
      "status": "ok",
      "response_time": 23,
      "last_check": "2024-01-01T00:00:00Z"
    },
    "redis": {
      "status": "ok",
      "response_time": 12,
      "last_check": "2024-01-01T00:00:00Z"
    }
  }
}

Error Responses​

All APIs return consistent error responses:

400 Bad Request​

{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid request data",
    "details": [
      {
        "field": "email",
        "message": "Invalid email format"
      }
    ]
  }
}

401 Unauthorized​

{
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Authentication required"
  }
}

403 Forbidden​

{
  "error": {
    "code": "FORBIDDEN",
    "message": "Insufficient permissions"
  }
}

404 Not Found​

{
  "error": {
    "code": "NOT_FOUND",
    "message": "Resource not found"
  }
}

500 Internal Server Error​

{
  "error": {
    "code": "INTERNAL_ERROR",
    "message": "An unexpected error occurred",
    "request_id": "uuid"
  }
}

Rate Limiting​

APIs are rate-limited to prevent abuse:

  • Authentication endpoints: 5 requests per minute per IP
  • Search endpoints: 100 requests per minute per user
  • General APIs: 1000 requests per hour per user

Rate limit headers are included in responses:

X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1640995200

Authentication​

Most endpoints require authentication via Bearer token:

Authorization: Bearer <jwt_token>

Some endpoints require specific roles:

  • Admin only: User management, system configuration
  • Moderator: Content moderation, user support
  • User: Personal data access, content creation

Pagination​

List endpoints support pagination with consistent parameters:

  • page: Page number (1-based, default: 1)
  • limit: Items per page (max: 100, default: 10)

Pagination info is included in responses:

{
  "pagination": {
    "page": 1,
    "limit": 10,
    "total": 150,
    "totalPages": 15,
    "hasNext": true,
    "hasPrev": false
  }
}

WebSocket Events​

Real-time updates are available via WebSocket connection at /api/ws.

Event Types​

  • event:created - New calendar event created
  • event:updated - Calendar event updated
  • event:deleted - Calendar event deleted
  • book:added - New book added to library
  • user:online - User came online
  • user:offline - User went offline

Event Format​

{
  "type": "event:created",
  "data": {
    "id": "uuid",
    "title": "New Event",
    "start": "2024-01-01T10:00:00Z"
  },
  "timestamp": "2024-01-01T09:30:00Z"
}
