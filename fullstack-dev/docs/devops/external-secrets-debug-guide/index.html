External Secrets Operator - Debug and Installation Guide

Overview​

This guide provides step-by-step instructions for installing, troubleshooting, and debugging External Secrets Operator on Google Kubernetes Engine (GKE) for the e-commerce platform.

Table of Contents​

  1. Installation Steps
  2. Verification and Health Checks
  3. Common Issues and Troubleshooting
  4. Configuration Examples
  5. Debugging Commands

Installation Steps​

1. Prerequisites​

Ensure you have the following tools installed:

  • kubectl configured with cluster access
  • helm v3.x installed
  • gcloud CLI configured

2. Connect to GKE Cluster​

# Connect to your GKE cluster
gcloud container clusters get-credentials sea1-prod-ecom-gke \
  --zone asia-southeast1-a \
  --project one-global-ecommerce-prod

3. Add External Secrets Helm Repository​

# Add the External Secrets Helm repository
helm repo add external-secrets https://charts.external-secrets.io

# Update Helm repositories
helm repo update

4. Install External Secrets Operator​

# Install External Secrets Operator
helm install external-secrets \
  external-secrets/external-secrets \
  -n external-secrets \
  --create-namespace

# Alternative: If you need to upgrade existing installation
helm upgrade external-secrets \
  external-secrets/external-secrets \
  -n external-secrets

Note: If you encounter "cannot re-use a name that is still in use" error, use the upgrade command instead.

Verification and Health Checks​

1. Check Installation Status​

# Verify Helm installation
helm list -n external-secrets

# Expected output:
# NAME                NAMESPACE           REVISION    UPDATED                 STATUS      CHART                   APP VERSION
# external-secrets    external-secrets    1           2025-10-16 11:48:22     deployed    external-secrets-0.x.x  v0.x.x

2. Check Pod Status​

# Check if all pods are running
kubectl get pods -n external-secrets

# Expected pods:
# - external-secrets-xxx (main controller)
# - external-secrets-cert-controller-xxx
# - external-secrets-webhook-xxx

3. Check Services and Endpoints​

# Verify services are created
kubectl get services -n external-secrets

# Check service endpoints
kubectl get endpoints -n external-secrets

4. Verify CRDs Installation​

# Check if Custom Resource Definitions are installed
kubectl get crd | grep external-secrets

# Expected CRDs:
# - externalsecrets.external-secrets.io
# - secretstores.external-secrets.io
# - clustersecretstores.external-secrets.io

Common Issues and Troubleshooting​

Issue 1: Installation Failed - Name Already in Use​

Error: INSTALLATION FAILED: cannot re-use a name that is still in use

Solution:

# Check existing installation
helm list -n external-secrets

# Option 1: Upgrade existing installation
helm upgrade external-secrets external-secrets/external-secrets -n external-secrets

# Option 2: Uninstall and reinstall
helm uninstall external-secrets -n external-secrets
helm install external-secrets external-secrets/external-secrets -n external-secrets --create-namespace

Issue 2: Pods Not Starting​

Symptoms: Pods stuck in Pending, ContainerCreating, or CrashLoopBackOff state

Debug Steps:

# Check pod events
kubectl describe pods -n external-secrets

# Check pod logs
kubectl logs -n external-secrets deployment/external-secrets
kubectl logs -n external-secrets deployment/external-secrets-cert-controller
kubectl logs -n external-secrets deployment/external-secrets-webhook

# Check resource constraints
kubectl top pods -n external-secrets
kubectl describe nodes

Issue 3: Webhook Issues​

Symptoms: ExternalSecrets not syncing, admission webhook errors

Debug Steps:

# Check webhook configuration
kubectl get validatingwebhookconfiguration | grep external-secrets
kubectl get mutatingwebhookconfiguration | grep external-secrets

# Check webhook service
kubectl get service -n external-secrets external-secrets-webhook

# Verify webhook certificates
kubectl get secret -n external-secrets

Issue 4: RBAC Permissions​

Symptoms: Permission denied errors in logs

Debug Steps:

# Check RBAC resources
kubectl get clusterrole | grep external-secrets
kubectl get clusterrolebinding | grep external-secrets
kubectl get serviceaccount -n external-secrets

# Verify permissions
kubectl auth can-i --list --as=system:serviceaccount:external-secrets:external-secrets

Configuration Examples​

1. Google Secret Manager SecretStore​

apiVersion: external-secrets.io/v1beta1
kind: SecretStore
metadata:
  name: gcpsm-secret-store
  namespace: prod-env
spec:
  provider:
    gcpsm:
      projectId: 'one-global-ecommerce-prod'
      auth:
        workloadIdentity:
          clusterLocation: asia-southeast1-a
          clusterName: sea1-prod-ecom-gke
          serviceAccountRef:
            name: external-secrets-sa

2. Sample ExternalSecret​

apiVersion: external-secrets.io/v1beta1
kind: ExternalSecret
metadata:
  name: app-secrets
  namespace: prod-env
spec:
  refreshInterval: 1h
  secretStoreRef:
    name: gcpsm-secret-store
    kind: SecretStore
  target:
    name: app-secrets
    creationPolicy: Owner
  data:
    - secretKey: database-url
      remoteRef:
        key: database-url
    - secretKey: jwt-secret
      remoteRef:
        key: jwt-secret

Debugging Commands​

1. General Health Check​

# Complete health check script
kubectl get all -n external-secrets
kubectl get crd | grep external-secrets
kubectl get events -n external-secrets --sort-by='.lastTimestamp'

2. Controller Logs​

# Main controller logs
kubectl logs -n external-secrets -l app.kubernetes.io/name=external-secrets -f

# Cert controller logs
kubectl logs -n external-secrets -l app.kubernetes.io/name=external-secrets-cert-controller -f

# Webhook logs
kubectl logs -n external-secrets -l app.kubernetes.io/name=external-secrets-webhook -f

3. Resource Status Check​

# Check SecretStore status
kubectl get secretstore -A
kubectl describe secretstore <secretstore-name> -n <namespace>

# Check ExternalSecret status
kubectl get externalsecret -A
kubectl describe externalsecret <externalsecret-name> -n <namespace>

# Check generated secrets
kubectl get secrets -A | grep external-secrets

4. Performance and Resource Monitoring​

# Resource usage
kubectl top pods -n external-secrets
kubectl describe limits -n external-secrets

# Network connectivity
kubectl exec -n external-secrets deployment/external-secrets -- nslookup secretmanager.googleapis.com
kubectl exec -n external-secrets deployment/external-secrets -- curl -I https://secretmanager.googleapis.com

Maintenance and Updates​

1. Regular Updates​

# Update Helm repository
helm repo update

# Check for new versions
helm search repo external-secrets/external-secrets

# Upgrade to latest version
helm upgrade external-secrets external-secrets/external-secrets -n external-secrets

2. Backup Configuration​

# Export current configuration
helm get values external-secrets -n external-secrets > external-secrets-values.yaml

# Backup custom resources
kubectl get secretstore -A -o yaml > secretstores-backup.yaml
kubectl get externalsecret -A -o yaml > externalsecrets-backup.yaml

3. Monitoring and Alerts​

# Check metrics endpoint (if enabled)
kubectl port-forward -n external-secrets service/external-secrets 8080:8080
curl http://localhost:8080/metrics

# Monitor events
kubectl get events -n external-secrets --watch

Best Practices​

  1. Resource Limits: Always set appropriate resource limits and requests
  2. RBAC: Follow principle of least privilege for service accounts
  3. Secrets Rotation: Implement regular secret rotation policies
  4. Monitoring: Set up monitoring and alerting for External Secrets components
  5. Backup: Regularly backup configurations and critical secrets
  6. Testing: Test secret synchronization in staging before production
