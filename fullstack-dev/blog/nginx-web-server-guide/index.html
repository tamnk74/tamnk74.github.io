Nginx: The High-Performance Web Server and Reverse Proxy

October 6, 2025 · 13 min read
Tam Nguyen
Full Stack Developer, Next.js for Production Creator

Nginx (pronounced "engine-x") is one of the most popular and powerful web servers in the world, powering over 30% of all websites on the internet. Originally created by Igor Sysoev in 2004 to solve the C10K problem (handling 10,000 concurrent connections), Nginx has evolved into a versatile tool that serves as a web server, reverse proxy, load balancer, and HTTP cache.

In this comprehensive guide, we'll explore what makes Nginx special, its key features, common use cases, and how to get started with basic configurations.

What is Nginx?​

Nginx is an open-source web server that can also function as a reverse proxy, load balancer, mail proxy, and HTTP cache. Unlike traditional web servers that create a new thread or process for each request, Nginx uses an event-driven, asynchronous architecture that allows it to handle thousands of concurrent connections with minimal resource usage.

Key Characteristics​

  • High Performance: Handles thousands of concurrent connections efficiently
  • Low Memory Usage: Uses significantly less memory compared to Apache
  • Event-Driven Architecture: Non-blocking I/O operations
  • Modular Design: Extensible through modules
  • Configuration Flexibility: Simple yet powerful configuration syntax

Core Features​

1. Web Server​

Nginx can serve static content (HTML, CSS, JavaScript, images) extremely efficiently. Its ability to handle static files makes it an excellent choice for serving modern web applications.

2. Reverse Proxy​

Acts as an intermediary between clients and backend servers, forwarding client requests to appropriate backend services and returning responses back to clients.

3. Load Balancing​

Distributes incoming requests across multiple backend servers to ensure optimal resource utilization and prevent any single server from becoming overwhelmed.

4. SSL/TLS Termination​

Handles SSL/TLS encryption and decryption, offloading this computationally intensive task from backend servers.

5. Caching​

Provides HTTP caching capabilities to store frequently requested content, reducing load on backend servers and improving response times.

Common Use Cases​

1. Static File Serving​

server {
    listen 80;
    server_name example.com;
    root /var/www/html;
    
    location / {
        try_files $uri $uri/ =404;
    }
    
    location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}

2. Reverse Proxy for Node.js Applications​

server {
    listen 80;
    server_name api.example.com;
    
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}

3. Load Balancing​

upstream backend {
    server backend1.example.com;
    server backend2.example.com;
    server backend3.example.com;
}

server {
    listen 80;
    server_name example.com;
    
    location / {
        proxy_pass http://backend;
    }
}

4. SSL/HTTPS Configuration​

server {
    listen 443 ssl http2;
    server_name example.com;
    
    ssl_certificate /path/to/certificate.crt;
    ssl_certificate_key /path/to/private.key;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:ECDHE-RSA-AES256-GCM-SHA384;
    
    location / {
        root /var/www/html;
        index index.html;
    }
}

# Redirect HTTP to HTTPS
server {
    listen 80;
    server_name example.com;
    return 301 https://$server_name$request_uri;
}

brew install nginx

macOS M4 Chip Specific Setup​

For Apple Silicon M4 processors, you may need to install Homebrew for ARM64 architecture if not already done:

# Install Homebrew (if not already installed)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Ensure you're using ARM64 architecture
arch
# Should output: arm64

# Install Nginx optimized for M4
brew install nginx

# Verify installation and architecture
file $(brew --prefix)/bin/nginx
# Should show: Mach-O 64-bit executable arm64

# Start Nginx with optimized settings for M4
brew services start nginx

# Check if running
brew services list | grep nginx

M4-Optimized Configuration​

For optimal performance on M4 chips, consider these configuration adjustments:

# /opt/homebrew/etc/nginx/nginx.conf (M4 optimized)
user _www;
worker_processes auto;  # M4 has efficient cores, auto-detection works well
error_log /opt/homebrew/var/log/nginx/error.log;

events {
    worker_connections 2048;  # M4 can handle higher connections efficiently
    use kqueue;  # Optimal for macOS
    multi_accept on;
}

http {
    include       mime.types;
    default_type  application/octet-stream;
    
    # M4 optimization settings
    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    
    # Buffer sizes optimized for M4 memory bandwidth
    client_body_buffer_size 128k;
    client_max_body_size 10m;
    client_header_buffer_size 1k;
    large_client_header_buffers 4 4k;
    output_buffers 1 32k;
    postpone_output 1460;
    
    server {
        listen 8080;  # Default Homebrew port
        server_name localhost;
        root /opt/homebrew/var/www;
        index index.html index.htm;
        
        location / {
            try_files $uri $uri/ =404;
        }
    }
}

Common M4 Considerations​

  1. File Paths: Homebrew on M4 installs to /opt/homebrew/ instead of /usr/local/
  2. Default Port: Homebrew Nginx typically runs on port 8080 to avoid permission issues
  3. Process Management: Use brew services for better integration with macOS
  4. Performance: M4's efficiency cores handle Nginx's event-driven architecture exceptionally well
# M4-specific paths
Configuration: /opt/homebrew/etc/nginx/nginx.conf
Document root: /opt/homebrew/var/www
Error log: /opt/homebrew/var/log/nginx/error.log
Access log: /opt/homebrew/var/log/nginx/access.log

Basic Configuration Structure​

Nginx configuration is typically located at /etc/nginx/nginx.conf and follows a hierarchical structure:

# Main context
user nginx;
worker_processes auto;
error_log /var/log/nginx/error.log;

# Events context
events {
    worker_connections 1024;
}

# HTTP context
http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;
    
    # Server context
    server {
        listen 80;
        server_name example.com;
        
        # Location context
        location / {
            root /var/www/html;
            index index.html;
        }
    }
}

Performance Optimization Tips​

1. Worker Processes​

Set worker processes to match your CPU cores:

worker_processes auto;

2. Worker Connections​

Increase worker connections for high-traffic sites:

events {
    worker_connections 2048;
    use epoll; # Linux
    use kqueue; # macOS/BSD
}

4. Gzip Compression​

Enable compression to reduce bandwidth:

gzip on;
gzip_vary on;
gzip_min_length 1024;
gzip_types text/plain text/css application/json application/javascript text/xml application/xml;

# M4 optimization: Use higher compression for better CPU/bandwidth trade-off
gzip_comp_level 6;  # M4 can handle higher compression efficiently
gzip_proxied any;

5. Browser Caching​

Set appropriate cache headers:

location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
    expires 1y;
    add_header Cache-Control "public, immutable";
}

Security Best Practices​

1. Hide Nginx Version​

server_tokens off;

2. Rate Limiting​

# Basic rate limiting by IP address
limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s;

# Rate limiting by user session (requires session ID)
limit_req_zone $cookie_sessionid zone=session:10m rate=5r/s;

# Rate limiting by server name (per domain)
limit_req_zone $server_name zone=domain:10m rate=100r/s;

# Rate limiting by request URI (specific endpoints)
limit_req_zone $request_uri zone=endpoint:10m rate=2r/s;

# Whitelist trusted IPs
geo $limit {
    default 1;
    10.0.0.0/8 0;     # Internal network
    192.168.0.0/16 0; # Private network
    127.0.0.1/32 0;   # Localhost
}

map $limit $limit_key {
    0 "";
    1 $binary_remote_addr;
}

limit_req_zone $limit_key zone=trusted:10m rate=20r/s;

# Different rate limits for different locations
location /api/ {
    limit_req zone=api burst=20 nodelay;
    limit_req_status 429;
    limit_req_log_level warn;
}

location /login {
    # Stricter rate limiting for login attempts
    limit_req zone=api burst=5 nodelay;
    limit_req_status 429;
}

location /search {
    # Higher burst for search with delay
    limit_req zone=api burst=50;
}

location /upload {
    # Very strict rate limiting for uploads
    limit_req_zone $binary_remote_addr zone=upload:10m rate=1r/m;
    limit_req zone=upload burst=2;
}

location /public/ {
    # Use trusted zone for public content
    limit_req zone=trusted burst=100 nodelay;
}

Advanced Rate Limiting Configurations​

Multiple Rate Limits:

# Apply multiple rate limits simultaneously
location /api/critical/ {
    limit_req zone=api burst=10 nodelay;
    limit_req zone=session burst=5 nodelay;
    limit_req zone=endpoint burst=2 nodelay;
}

Dynamic Rate Limiting Based on Response Status:

# Stricter limits after failed attempts
map $status $loggable {
    ~^[23] 0;  # Don't log successful requests
    default 1;
}

limit_req_zone $binary_remote_addr zone=strict:10m rate=1r/s;

location /api/auth {
    limit_req zone=api burst=10;
    
    # Apply stricter limits if previous requests failed
    if ($http_x_failed_attempts) {
        limit_req zone=strict burst=3;
    }
}

Rate Limiting with Custom Error Pages:

# Custom error page for rate limited requests
error_page 429 /rate_limit.html;

location = /rate_limit.html {
    root /var/www/error;
    internal;
    add_header Content-Type text/html;
    add_header Retry-After 60;
}

location /api/ {
    limit_req zone=api burst=20 nodelay;
    limit_req_status 429;
}

Burst Control Strategies:

# No burst - strict limiting
location /strict-api/ {
    limit_req zone=api;
}

# Burst with delay - smoother for users
location /smooth-api/ {
    limit_req zone=api burst=10;
}

# Burst with nodelay - fast response but strict enforcement
location /fast-api/ {
    limit_req zone=api burst=10 nodelay;
}

3. Security Headers​

add_header X-Frame-Options "SAMEORIGIN" always;
add_header X-XSS-Protection "1; mode=block" always;
add_header X-Content-Type-Options "nosniff" always;
add_header Referrer-Policy "no-referrer-when-downgrade" always;
add_header Content-Security-Policy "default-src 'self'" always;

Comprehensive Security Headers Explained​

X-Frame-Options - Prevents Clickjacking Attacks:

# DENY: Prevents any domain from framing the content
add_header X-Frame-Options "DENY" always;

# SAMEORIGIN: Only allows framing from the same origin
add_header X-Frame-Options "SAMEORIGIN" always;

# ALLOW-FROM: Allows framing from specific domains
add_header X-Frame-Options "ALLOW-FROM https://trusted-site.com" always;

Content-Security-Policy (CSP) - Prevents XSS and Data Injection:

# Basic CSP - only allow resources from same origin
add_header Content-Security-Policy "default-src 'self'" always;

# Advanced CSP for modern web applications
add_header Content-Security-Policy "
    default-src 'self';
    script-src 'self' 'unsafe-inline' https://cdn.jsdelivr.net https://ajax.googleapis.com;
    style-src 'self' 'unsafe-inline' https://fonts.googleapis.com;
    font-src 'self' https://fonts.gstatic.com;
    img-src 'self' data: https:;
    connect-src 'self' https://api.example.com;
    frame-ancestors 'none';
    base-uri 'self';
    form-action 'self';
    upgrade-insecure-requests;
" always;

# CSP for Single Page Applications (React/Vue/Angular)
add_header Content-Security-Policy "
    default-src 'self';
    script-src 'self' 'sha256-xyz123...' 'unsafe-eval';
    style-src 'self' 'unsafe-inline';
    img-src 'self' data: blob: https:;
    font-src 'self' data:;
    connect-src 'self' wss: https:;
    worker-src 'self' blob:;
" always;

Strict-Transport-Security (HSTS) - Enforces HTTPS:

# Basic HSTS - 1 year, include subdomains
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

# HSTS with preload (for submission to browsers' preload lists)
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload" always;

# Development/staging environment (shorter duration)
add_header Strict-Transport-Security "max-age=300; includeSubDomains" always;

X-Content-Type-Options - Prevents MIME Type Sniffing:

# Prevents browsers from interpreting files as different MIME types
add_header X-Content-Type-Options "nosniff" always;

Referrer-Policy - Controls Referrer Information:

# No referrer when protocol security is downgraded
add_header Referrer-Policy "no-referrer-when-downgrade" always;

# Strict referrer policy options
add_header Referrer-Policy "strict-origin-when-cross-origin" always;
add_header Referrer-Policy "same-origin" always;
add_header Referrer-Policy "no-referrer" always;

# Origin only (no path information)
add_header Referrer-Policy "origin" always;

X-XSS-Protection - Legacy XSS Protection:

# Enable XSS filtering and block page rendering if attack detected
add_header X-XSS-Protection "1; mode=block" always;

# Note: This header is largely deprecated in favor of CSP
# Modern browsers rely on Content-Security-Policy instead

Permissions-Policy - Controls Browser Features:

# Disable potentially dangerous features
add_header Permissions-Policy "
    camera=(),
    microphone=(),
    geolocation=(),
    payment=(),
    usb=(),
    magnetometer=(),
    gyroscope=(),
    speaker=()
" always;

# Allow specific features for your domain only
add_header Permissions-Policy "
    camera=(self),
    microphone=(self),
    geolocation=(self 'https://trusted-maps.com'),
    payment=(self)
" always;

Cross-Origin-Embedder-Policy & Cross-Origin-Opener-Policy:

# Enable cross-origin isolation for advanced features
add_header Cross-Origin-Embedder-Policy "require-corp" always;
add_header Cross-Origin-Opener-Policy "same-origin" always;

# More permissive options
add_header Cross-Origin-Embedder-Policy "credentialless" always;
add_header Cross-Origin-Opener-Policy "same-origin-allow-popups" always;

Complete Security Headers Configuration​

Production-Ready Security Headers Set:

# Complete security headers configuration
location / {
    # HSTS - Force HTTPS for 1 year
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload" always;
    
    # Prevent clickjacking
    add_header X-Frame-Options "SAMEORIGIN" always;
    
    # Prevent MIME type sniffing
    add_header X-Content-Type-Options "nosniff" always;
    
    # Control referrer information
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;
    
    # Comprehensive CSP
    add_header Content-Security-Policy "
        default-src 'self';
        script-src 'self' 'unsafe-inline' https://cdn.jsdelivr.net;
        style-src 'self' 'unsafe-inline' https://fonts.googleapis.com;
        font-src 'self' https://fonts.gstatic.com;
        img-src 'self' data: https:;
        connect-src 'self' https:;
        frame-ancestors 'none';
        base-uri 'self';
        form-action 'self';
        upgrade-insecure-requests;
    " always;
    
    # Control browser features
    add_header Permissions-Policy "
        camera=(),
        microphone=(),
        geolocation=(),
        payment=(),
        usb=()
    " always;
    
    # Cross-origin policies
    add_header Cross-Origin-Embedder-Policy "credentialless" always;
    add_header Cross-Origin-Opener-Policy "same-origin-allow-popups" always;
    
    # Remove server information
    add_header X-Powered-By "" always;
    
    # Custom security headers
    add_header X-Robots-Tag "noindex, nofollow" always;  # For admin areas
    add_header Cache-Control "no-store, no-cache, must-revalidate" always;  # For sensitive pages
}

Environment-Specific Configurations​

Development Environment:

# Relaxed headers for development
add_header Content-Security-Policy "
    default-src 'self' 'unsafe-inline' 'unsafe-eval';
    connect-src 'self' ws: wss: http: https:;
" always;

add_header Strict-Transport-Security "max-age=300" always;  # Short duration

API Endpoints:

location /api/ {
    # API-specific security headers
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-Frame-Options "DENY" always;
    add_header Cache-Control "no-store" always;
    add_header Pragma "no-cache" always;
    
    # CORS headers (if needed)
    add_header Access-Control-Allow-Origin "https://yourdomain.com" always;
    add_header Access-Control-Allow-Methods "GET, POST, PUT, DELETE, OPTIONS" always;
    add_header Access-Control-Allow-Headers "Authorization, Content-Type" always;
    add_header Access-Control-Max-Age "86400" always;
}

Static Assets:

location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
    # Relaxed CSP for static assets
    add_header Content-Security-Policy "default-src 'self'" always;
    add_header Cache-Control "public, max-age=31536000, immutable" always;
    add_header X-Content-Type-Options "nosniff" always;
}

Modern Web Development with Nginx​

Microservices Architecture​

Nginx excels in microservices environments, routing requests to different services based on URL patterns:

location /auth/ {
    proxy_pass http://auth-service:3001/;
}

location /users/ {
    proxy_pass http://user-service:3002/;
}

location /orders/ {
    proxy_pass http://order-service:3003/;
}

API Gateway​

Use Nginx as an API gateway for request routing, authentication, and rate limiting:

location /api/v1/ {
    auth_request /auth;
    proxy_pass http://api-backend/;
}

location = /auth {
    internal;
    proxy_pass http://auth-service/verify;
    proxy_pass_request_body off;
    proxy_set_header Content-Length "";
    proxy_set_header X-Original-URI $request_uri;
}

Monitoring and Logging​

Access Logs​

log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                '$status $body_bytes_sent "$http_referer" '
                '"$http_user_agent" "$http_x_forwarded_for"';

access_log /var/log/nginx/access.log main;

Error Logs​

error_log /var/log/nginx/error.log warn;

Status Module​

Enable the status module for monitoring:

location /nginx_status {
    stub_status on;
    access_log off;
    allow 127.0.0.1;
    deny all;
}

Nginx has proven itself as an indispensable tool in modern web infrastructure. Its performance, flexibility, and rich feature set make it suitable for everything from simple static websites to complex, high-traffic applications. As web applications continue to evolve, Nginx remains at the forefront, adapting to new requirements while maintaining its core strengths: high performance, low resource usage, and exceptional reliability.
