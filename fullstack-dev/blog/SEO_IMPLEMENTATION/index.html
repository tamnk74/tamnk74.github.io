SEO Implementation Guide

October 1, 2025 · 5 min read

This document outlines the comprehensive SEO implementation added to the Next.js application using next-sitemap and next-seo packages.

📋 Overview​

The SEO implementation includes:

  • Sitemap Generation with next-sitemap
  • Meta Tags Management with next-seo
  • Structured Data (JSON-LD) support
  • Multilingual SEO for English and Vietnamese
  • Social Media Optimization (OpenGraph, Twitter Cards)
  • SEO Health Monitoring tools

🗂️ File Structure​

src/
├── config/
│   ├── seo.ts              # Default SEO configuration
│   └── seo-pages.ts        # Page-specific SEO configurations
├── components/
│   ├── SEOWrapper.tsx      # Global SEO wrapper component
│   └── PageSEO.tsx         # Page-specific SEO components
├── hooks/
│   └── useSEO.ts          # SEO utility hooks
└── app/
    └── [locale]/
        └── layout.tsx      # Updated with SEO integration

Configuration Files:
├── next-sitemap.config.js  # Sitemap generation configuration
└── scripts/
    └── seo-health-check.js # SEO monitoring script

🔧 Core Components​

1. Default SEO Configuration (src/config/seo.ts)​

export const defaultSEO: DefaultSeoProps = {
  titleTemplate: '%s | Next.js Production App',
  defaultTitle: 'Next.js Production App',
  description: 'A production-ready Next.js application with modern features...',
  openGraph: {
    type: 'website',
    locale: 'en_US',
    url: 'https://next-for-prod.com/',
    siteName: 'Next.js Production App',
    images: [{ url: '/og-default.jpg', width: 1200, height: 630 }]
  },
  twitter: {
    handle: '@nextjsprod',
    site: '@nextjsprod',
    cardType: 'summary_large_image'
  }
};

2. Page-Specific SEO (src/config/seo-pages.ts)​

Provides localized SEO configurations for all major pages:

  • Home, About, Libraries, Tools, Calendar, Posts, Feedback
  • Authentication pages (Login, Signup)
  • Multilingual support (English/Vietnamese)
  • Canonical URLs and language alternates

3. SEO Components (src/components/PageSEO.tsx)​

// Specialized components for each page type
<HomeSEO />
<AboutSEO />
<LibsSEO />
<ArticleSEO 
  title="Article Title"
  description="Article description"
  url="https://example.com/article"
  datePublished="2024-01-01"
  image="/article-image.jpg"
/>

4. SEO Hooks (src/hooks/useSEO.ts)​

// Automatic SEO based on current route
const seoConfig = useSEO();

// Manual page specification
const seoConfig = useSEO('home');

// Get canonical URL
const canonicalUrl = useCanonicalUrl();

// Generate structured data
const organizationData = useStructuredData('organization');

🗺️ Sitemap Configuration​

next-sitemap.config.js​

module.exports = {
  siteUrl: process.env.SITE_URL || 'https://yourdomain.com',
  generateRobotstxt: true,
  sitemapSize: 7000,
  changefreq: 'daily',
  priority: 0.7,
  additionalPaths: async (config) => {
    // 29 URLs total with multilingual support
    // Static pages, dynamic routes, auth pages
  },
  robotsTxtOptions: {
    policies: [
      { userAgent: '*', allow: '/' },
      { userAgent: 'GPTBot', disallow: ['/'] },
      { userAgent: 'ChatGPT-User', disallow: ['/'] },
      { userAgent: 'CCBot', disallow: ['/'] }
    ]
  }
};

Generated Files​

  • public/sitemap.xml: 29 URLs with proper priorities and change frequencies
  • public/robots.txt: AI crawler blocking and sitemap references

🌐 Multilingual SEO​

Each page supports both English and Vietnamese with:

  • Localized titles and descriptions
  • Proper hreflang attributes
  • Language-specific OpenGraph images
  • Canonical URLs for each locale
// English
canonical: `${BASE_URL}/en/about`
hreflang: [
  { hrefLang: 'en', href: `${BASE_URL}/en/about` },
  { hrefLang: 'vi', href: `${BASE_URL}/vi/about` },
  { hrefLang: 'x-default', href: `${BASE_URL}/en/about` }
]

// Vietnamese  
canonical: `${BASE_URL}/vi/about`
locale: 'vi_VN'

📊 Structured Data Support​

Organization Schema​

  "@context": "https://schema.org",

  "name": "Next.js Production App",
  "url": "https://next-for-prod.com",

🚀 Usage Examples​

1. Basic Page SEO​

import { HomeSEO } from '@/components/PageSEO';

export default function HomePage() {
  return (
    <>
      <HomeSEO />
      <main>
        {/* Page content */}
      </main>
    </>
  );
}

2. Custom SEO Override​

import { PageSEO } from '@/components/PageSEO';

export default function CustomPage() {
  return (
    <>
      <PageSEO 
        pageKey="home"
        customSEO={{
          title: 'Custom Title',
          description: 'Custom description'
        }}
      />
      <main>
        {/* Page content */}
      </main>
    </>
  );
}

3. Article with Structured Data​

import { ArticleSEO } from '@/components/PageSEO';

export default function BlogPost() {
  return (
    <>
      <ArticleSEO
        title="How to Build Production-Ready Next.js Apps"
        description="Complete guide to building scalable Next.js applications"
        url="https://next-for-prod.com/en/posts/production-nextjs"
        datePublished="2024-01-15"
        authorName="Development Team"
        image="/posts/production-nextjs-og.jpg"
      />
      <article>
        {/* Article content */}
      </article>
    </>
  );
}

  • ✅ Sitemap accessibility
  • ✅ Robots.txt functionality
  • ✅ Page response codes
  • ✅ Meta tag presence
  • ✅ OpenGraph data
  • ✅ Structured data validation

🎯 Best Practices Implemented​

  1. Title Templates: Consistent branding across all pages
  2. Unique Descriptions: Each page has specific, descriptive meta descriptions
  3. Image Optimization: 1200x630 OpenGraph images for optimal sharing
  4. Mobile Optimization: Proper viewport and mobile meta tags
  5. Accessibility: Alt text for images, proper heading structure
  6. Performance: Minimal SEO overhead, efficient component structure
  7. Internationalization: Proper language declarations and alternates
