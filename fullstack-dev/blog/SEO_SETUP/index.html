SEO Setup with next-sitemap

October 1, 2025 · 3 min read

This project uses next-sitemap to generate SEO-optimized sitemaps and robots.txt files.

Features​

✅ Automatic Sitemap Generation: Generates comprehensive sitemaps for all public routes
✅ Multi-language Support: Full support for English (en) and Vietnamese (vi) locales
✅ Robots.txt Generation: Automatic robots.txt with AI crawler blocking
✅ SEO Optimization: Proper priorities, change frequencies, and hreflang attributes
✅ Build Integration: Automatically runs after each build

Configuration​

Environment Variables​

Set your production domain in your environment:

# .env.local or .env.production
NEXT_PUBLIC_APP_URL=https://next-for-prod.com

Routes Included​

The sitemap includes these public routes for both locales:

  • Homepage: /en, /vi (Priority: 1.0, Daily updates)
  • Content Pages: /about, /libs, /posts, /calendar (High priority)
  • Tools: /tools, /search-*, /roadmap/* (Medium priority)
  • Auth Pages: /login, /signup (Lower priority)
  • Utility Pages: /feedback (Medium priority)

Routes Excluded​

These routes are excluded from search engines:

  • Admin pages (/admin/*)
  • User profiles (/profile/*, /me/*)
  • Private settings (/settings)
  • API routes (/api/*)
  • Test pages (/test-*)
  • Authentication callbacks (/auth/*)
  • Edit/create forms

Usage​

Generate Sitemap​

# Manual generation
pnpm sitemap:generate

# Automatic generation after build
pnpm build  # Runs next-sitemap via postbuild script

Generated Files​

  • public/sitemap.xml - Main sitemap with all URLs
  • public/robots.txt - Search engine directives

SEO Utilities​

The project includes SEO utilities in src/utils/seo.ts:

import { generatePageSEO } from '@/utils/seo';

export const metadata = generatePageSEO('home', 'en');

Available functions:

  • generateSEOMetadata() - Comprehensive metadata generation
  • generatePageSEO() - Page-specific SEO metadata
  • generateStructuredData() - JSON-LD structured data

AI Crawler Blocking​

The robots.txt automatically blocks these AI crawlers:

  • GPTBot
  • ChatGPT-User
  • CCBot
  • anthropic-ai
  • Claude-Web

Monitoring​

Google Search Console​

  1. Submit your sitemap: https://yourdomain.com/sitemap.xml
  2. Monitor indexing status
  3. Check for crawl errors

Validation Tools​

  • Google Rich Results Test
  • Bing Webmaster Tools
  • XML Sitemap Validator

Configuration File​

Edit next-sitemap.config.js to:

  • Add new routes
  • Modify priorities and change frequencies
  • Update hreflang mappings
  • Exclude additional paths

Best Practices​

  1. Update sitemap after major changes: Run pnpm sitemap:generate
  2. Monitor crawl budget: Use robots.txt to guide crawlers to important pages
  3. Regular audits: Check for 404s and crawl errors
  4. Performance: Exclude dynamic/user-specific pages from sitemap
  5. Localization: Ensure hreflang attributes are correct for international SEO

Troubleshooting​

Sitemap not updating​

  • Run pnpm build to regenerate
  • Check next-sitemap.config.js for syntax errors
  • Verify environment variables

Routes missing from sitemap​

  • Add to additionalPaths in config
  • Check exclusion rules
  • Ensure pages are statically accessible

robots.txt issues​

  • Verify generateRobotsTxt: true in config
  • Check policy rules for conflicts
  • Test with robots.txt validators
