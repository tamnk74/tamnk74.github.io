Elasticsearch Full-Text Search Setup

October 1, 2025 · 5 min read

This guide will help you set up Elasticsearch for full-text search functionality in your Next.js application.

Prerequisites​

  • Node.js and npm/yarn installed
  • Next.js application running
  • Supabase database with posts

1. Install Elasticsearch​

Option A: Using Docker (Recommended for development)​

# Start Elasticsearch with Docker
docker run -d \
  --name elasticsearch \
  -p 9200:9200 \
  -e "discovery.type=single-node" \
  -e "xpack.security.enabled=false" \
  elasticsearch:8.11.0

# Verify Elasticsearch is running
curl http://localhost:9200

Option B: Local Installation​

  1. Download Elasticsearch from elastic.co
  2. Follow the installation guide for your operating system
  3. Start Elasticsearch service

2. Environment Variables​

Add these environment variables to your .env.local file:

# Elasticsearch Configuration
ELASTICSEARCH_URL=http://localhost:9200
ELASTICSEARCH_POSTS_INDEX=posts

# Optional: Authentication (if your Elasticsearch requires auth)
# ELASTICSEARCH_USERNAME=your_username
# ELASTICSEARCH_PASSWORD=your_password
# ELASTICSEARCH_APIKEY=your_api_key

3. Initialize the Index​

Option A: Using the Web Interface (Development)​

  1. Go to your posts page (/posts)
  2. In the debug section, click "Check Status" to verify Elasticsearch connection
  3. Click "Initialize Index" to create the index and bulk import existing posts

Option B: Using the Setup Script​

# Run the setup script
npx tsx scripts/setup-elasticsearch.ts

# Or add to package.json scripts:
# "setup:elasticsearch": "tsx scripts/setup-elasticsearch.ts"
# Then run: npm run setup:elasticsearch

Option C: Using the API Endpoint​

# Check Elasticsearch status
curl http://localhost:3000/api/elasticsearch/init

# Initialize the index
curl -X POST http://localhost:3000/api/elasticsearch/init

4. Features​

Search Functionality​

  • Full-text search: Search across post titles and content
  • Fuzzy matching: Handles typos and similar words
  • Highlighting: Search terms are highlighted in results
  • Fallback: Falls back to database search if Elasticsearch is unavailable
  • Responsive: Search bar available in navbar on desktop and mobile

Search Features​

  • Keyboard shortcut: Press Cmd+K (Mac) or Ctrl+K (Windows/Linux) to focus search
  • Real-time results: Search results update as you type
  • Pagination: Large result sets are paginated
  • Sorting: Results sorted by relevance score and creation date

5. API Endpoints​

Search Posts​

GET /api/posts/search?q=search_term&page=1&limit=10

Initialize Elasticsearch​

GET /api/elasticsearch/init    # Check status
POST /api/elasticsearch/init   # Initialize index

Sync Posts to Elasticsearch​

GET /api/elasticsearch/sync     # Health check and endpoints info
POST /api/elasticsearch/sync?action=sync   # Sync all posts
POST /api/elasticsearch/sync?action=health # Health check only

6. Automatic Indexing​

Posts are automatically indexed in Elasticsearch when:

  • Creating a post: New posts are indexed immediately after successful creation
  • Updating a post: Modified posts are re-indexed with the latest content
  • Deleting a post: Posts are removed from the Elasticsearch index

The indexing happens automatically in the background and won't affect the user experience. If Elasticsearch is unavailable, the database operations will still succeed, and only a warning will be logged.

Manual Sync​

If you need to sync existing posts or re-index everything:

Using the API​

# Check Elasticsearch health
curl http://localhost:3000/api/elasticsearch/sync

# Sync all posts to Elasticsearch
curl -X POST "http://localhost:3000/api/elasticsearch/sync?action=sync"

# Health check only
curl -X POST "http://localhost:3000/api/elasticsearch/sync?action=health"

Using the Sync Service​

import { syncPostsToElasticsearch } from '@/shared/services/sync';

const result = await syncPostsToElasticsearch();
console.log(`Synced ${result.syncedCount} posts`);

Monitoring Indexing​

During development, you can monitor indexing in the browser console:

  • ✅ Post indexed in Elasticsearch: 123
  • ✅ Post updated in Elasticsearch: 123
  • ✅ Post deleted from Elasticsearch: 123
  • ⚠️ Failed to index post in Elasticsearch: [error details]

7. Troubleshooting​

Elasticsearch not connecting​

  1. Verify Elasticsearch is running: curl http://localhost:9200
  2. Check environment variables in .env.local
  3. Check Docker container status: docker ps
  4. View Docker logs: docker logs elasticsearch

Search not working​

  1. Check if index exists: Use the debug panel in /posts
  2. Verify posts are indexed: Look for "posts indexed" message
  3. Check browser console for JavaScript errors
  4. Verify API endpoints are working

Performance Issues​

  1. Increase Elasticsearch memory: Add -e "ES_JAVA_OPTS=-Xms512m -Xmx512m" to Docker command
  2. Optimize search queries in elasticsearch.ts
  3. Implement caching for frequently searched terms

8. Production Deployment​

Elasticsearch Service​

For production, consider using:

  • Elastic Cloud: Managed Elasticsearch service
  • AWS OpenSearch: Amazon's Elasticsearch service
  • Self-hosted: Deploy Elasticsearch on your own infrastructure

Environment Variables for Production​

ELASTICSEARCH_URL=https://your-production-elasticsearch-url
ELASTICSEARCH_APIKEY=your-production-api-key
ELASTICSEARCH_POSTS_INDEX=production_posts

Security Considerations​

  1. Enable authentication on Elasticsearch
  2. Use HTTPS for Elasticsearch connections
  3. Restrict network access to Elasticsearch
  4. Regularly update Elasticsearch version

9. Monitoring and Maintenance​

Health Checks​

The application includes health checks:

  • Elasticsearch availability check in search API
  • Fallback to database search if Elasticsearch is down
  • Status indicators in development debug panel

Index Maintenance​

  • Reindexing: Use the initialize endpoint to rebuild the index
  • Mapping updates: Update the mapping in elasticsearch.ts and reinitialize
  • Performance monitoring: Monitor query performance and optimize as needed

10. Customization​

Search Configuration​

Edit src/shared/services/elasticsearch.ts to customize:

  • Search fields and boost values
  • Highlighting settings
  • Fuzzy matching sensitivity
  • Result sorting criteria

UI Customization​

Edit components to customize:

  • src/shared/ui/search-bar/index.tsx: Search bar appearance
  • src/features/posts/list/ui.tsx: Results display
  • src/app/[locale]/components/Navbar.tsx: Navbar integration

Support​

If you encounter issues:

  1. Check the troubleshooting section above
  2. Review Elasticsearch logs
  3. Check browser console for errors
  4. Verify environment variables are set correctly
